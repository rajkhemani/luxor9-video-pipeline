| | Track A — The Platform | Track B — The Nine |

Track A is the engine — real deployments and public traction make Track B's scarcity credible rather than hollow. The Brand Guardian's core job is preventing **track bleed**: no urgency or comparison language in Track B; no faux-exclusivity theater in Track A.

## 3. ASSET PRODUCTION MATRIX (OpenMontage)

All video assets are produced through the OpenMontage pipeline system in this repository — per Rule Zero, **no ad-hoc generation**. Each production run goes through preflight (provider menu), proposal with runtime selection, and human approval **before any paid generation call**.

| # | Asset | Track | Pipeline | Style / treatment | Target |
|---|-------|-------|----------|-------------------|--------|
| V1 | Launch ad — 7 platform variants | A | `social-creative` | `flat-motion-graphics`; agent-hierarchy execution trace as hero visual | Day 22 |
| V2 | Agent-hierarchy explainer (60–90s) | A | `animated-explainer` | `clean-professional`; Commander → Workers decomposition animated | Day 14 |
| V3 | Product demo — terminal + dashboard walkthrough | A | `screen-demo` | Synthetic TerminalScene; deterministic, privacy-safe | Day 10 |
| V4 | Founder/spokesperson launch message | A | `avatar-spokesperson` | Clean, direct-to-camera | Day 22 |
| V5 | Genesis hero loop (20s seamless, Obsidian → Gold) | B | `cinematic` | Per Phase 1 UI/UX spec: particle formation → cipher → awakening | Day 30 |
| V6 | The Nine brand film teaser | B | `cinematic` | Cricketer Editorial DNA; chiaroscuro, gold rim light | Day 42 |
| V7 | Patron quarterly film #1 (15 min) | B | `cinematic` + `hybrid` | Factory-evolution documentary; patron-only | Day 70 |

**Production governance (binding):**
1. Each asset starts with preflight + `provider_menu_summary()` — capabilities presented before promises are made.
2. Composition runtime (Remotion vs HyperFrames vs FFmpeg) is presented as options and locked at proposal — never silently defaulted.
3. Motion-required assets (V1, V5, V6) must not degrade to still-image fallback without explicit approval.
4. Music plan surfaced at proposal for every asset (library → API → user-provided → none).
5. Asset generation begins only after production-plan approval.
